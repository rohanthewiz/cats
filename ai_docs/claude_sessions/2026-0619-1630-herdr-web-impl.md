## Feasibility findings

- herdr is already a **server/client** app (~268K lines Rust). The **server owns PTYs, all
  app state, agent detection, layout, and rendering**; it renders the entire UI into a
  **semantic cell grid** shipped over a wire protocol. The client is a thin frame blitter.
- The wire protocol (`src/protocol/wire.rs`) is the natural seam between Go and Rust.
- `go-libghostty` binds the **same libghostty-vt** herdr already vendors, but it's CGO,
  no releases, no API stability, and **still needs libghostty-vt built with Zig** — so it
  does **not** dodge the Zig toolchain. It is **not needed in Phase A** (the Rust server
  already does VT emulation; the browser just draws finished cells). It becomes the
  cornerstone only in Phase B.

### Migration plan (saved to `~/.claude/plans/deep-jingling-fog.md`)

- **Phase A** — Go/rweb gateway on the unchanged Rust server (browser herdr). *(built this session)*
- **Phase B** — Go owns PTY + VT via go-libghostty (needs Zig); retires `src/pty`, `src/pane`, `src/ghostty`, `src/terminal`.
- **Phase C** — port portable logic (app state, BSP layout, detection, session) to Go; retire the Rust core.
- **Phase D** — decommission Rust.
- Honest caveat: hybrid temporarily means maintaining **both** stacks; net maintenance only drops once B/C retire Rust subsystems.

## Key technical findings (non-obvious)

- **Wire format:** `[u32-LE length][bincode v2 "standard" payload]`, max 2 MiB. bincode
  standard = little-endian + variable-int encoding; enum discriminant = varint u32;
  `Option<T>` = 1-byte 0/1 tag; `Vec`/`String` = varint length + bytes. Hand-wrote the codec
  in Go (`internal/wire/codec.go`) — no external bincode lib needed.
- **Protocol version mismatch:** local checkout was stale at `PROTOCOL_VERSION = 13`; the
  installed **herdr 0.7.0 binary speaks protocol 14**. Had to `git fetch origin` to get the
  real source (tag `v0.7.0` / `origin/master`). Target **14**.
- **Proto-14 enum shift:** `ServerMessage::WindowTitle` was inserted at index **7**, pushing
  `ReloadSoundConfig`→8 and `MouseCapture`→9 (vs proto 13). That explained the mysterious
  "kind 9, 2 bytes" message seen early on (= `MouseCapture { enabled: bool }`).
  `ClientMessage` variants unchanged, so Hello/Input/Resize/Detach were already correct.
- **Isolated test server is hard with the 0.7.0 binary:** it ignores `HERDR_SESSION`,
  `XDG_CONFIG_HOME`, and `HOME` for socket/session resolution (resolves the real
  `~/.config/herdr` via the system home API) and refuses to start a 2nd server. An isolated
  server needs a **source build**, which is blocked: `zig` is not installed; `build.rs` runs
  `zig build`; macOS SDK is a fresh 26.5 (the exact Zig/SDK risk the user flagged). `zig 0.16`
  is available via brew when Phase B needs it.
- **Workaround for testing:** attach **read-only** to the live default-session client socket
  (`~/.config/herdr/herdr-client.sock`); herdr is tmux-like and renders **per-client**, so a
  brief attach/detach doesn't resize other clients. Never injected input into the live session.

### Phase A (core) — verified

   server proto 14, no error`, `FrameData` decodes into the real herdr UI (sidebar, tab bar,
   Claude Code content).

3. Full chain browser↔gateway↔herdr proven headlessly: a real `110×30` frame (3300 cells,
   cursor, ~59 KB JSON) streamed over WebSocket and rendered at the web client's size.

### Phase A polish — verified

- **Frame diffing:** first message = full frame (~53 KB), then only changed cells
  (`fdiff`). Measured **~100 B per steady-state update (~500× cut)**. Browser keeps a cell
  buffer and patches by index; redraws coalesced via `requestAnimationFrame`.
- **Mouse input (SGR 1006):** browser sends click/drag/wheel, **gated on server
  `MouseCapture`** (decoded correctly as kind 9). Context-menu suppressed while captured.
- **Clipboard:** herdr→browser copy (OSC 52, base64 → `navigator.clipboard`); browser→herdr
  paste via structured `InputEvents::Paste` (server applies bracketed paste correctly).
- **OSC 8 hyperlinks:** underlined + click-to-open when mouse not captured.
- **Window title** → `document.title`; **notify toasts** UI.
- Probe tally confirmed: `frame:1, fdiff:6, mouse:1`; `mouse capture: true` decoded.
- Deferred: Kitty graphics passthrough.

**Verification boundary:** all decode/forward plumbing confirmed read-only against the live
server. Browser→herdr interaction (clicks/drags/paste/hyperlink opens) is coded and compiles
but was deliberately **not injected** into the live session.

## How to run

```bash
cd ~/projs/go/herdr-web
go build ./...

# Browser herdr (attaches to the live default session — keystrokes reach its focused pane):
go run ./cmd/gateway --addr :8420     # open http://localhost:8420
# stop: pkill -f herdr-gateway

# Headless protocol checks (read-only):
go run ./cmd/smoke   --socket ~/.config/herdr/herdr-client.sock --frames 2
go run ./cmd/wsprobe --msgs 8         # full frame + diffs + control messages
```

Gateway was left running on `:8420` at end of session.
