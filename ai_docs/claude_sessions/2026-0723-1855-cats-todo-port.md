## What changed

### 1. §7 enablers (`internal/app`, `cmd/catway`, `internal/browserproto`)

Two API gaps blocked the port; both fixed server-side:

- **`tab.create` returns `TabCreateResult{num, pane}`** — the new tab's public
  number and its root pane id. `Session.CreateTab` switches to the new tab, so
  the globally focused pane after `ApplyModel` *is* the root pane; the
  dispatcher just reads `FocusedPane()`. Lets a client chain straight into
  `send_input`/`wait_for_output` without diffing `pane.list`.
- **`PaneInfo` gains an embedded `PaneMeta` block** (`agent`, `agent_state`,
  `title`, `cwd`, all omitempty) on `pane.list`/`pane.get`. The session can't
  know runtime state, so a new `Backend.PaneMeta(pane) PaneMeta` seam supplies
  it; the dispatcher merges it into query results. catway answers from the
  per-pane runtime cache (`rt.effectiveAgent()` — the same hook-vs-detection
  arbitration the sidebar shows — plus `rt.title`/`rt.cwd`). Unknown pane ⇒
  zero value, no ok flag.
- `browserproto/cmd.go` re-exports `TabCreateResult` + `PaneMeta`.
- Tests: `fakeBackend.PaneMeta` (canned `paneMeta` map),
  `TestDispatchTabCreateResult`, `TestDispatchPaneMetaMerge`.

### 2. The port (`cmd/cats-todo`, new)

Near-verbatim carries: `fuzzylist.go` (+ its tests), `styles.go`, `util.go`,
`store.go`, most of `ui.go`, `cli.go` (the `add` subcommand). What actually
changed:

- **`client.go`** replaces herdr's JSON-RPC `herdr.go`: `catsClient` over
  `ctlproto.Call` (socket via `ctlproto.ResolveSocket` / `CATS_CONTROL_SOCKET`;
  constructor pings and degrades to nil — backlog management works socketless,
  only drops need it). Polling loops became server-side waits:
  - shell-ready = `wait_for_output` regex `\S` (seeded with current screen);
  - command echo = substring wait; claude-ready = the probe list folded into
    one `regexp.QuoteMeta`-joined alternation. All best-effort.
  - `runCommand` keeps herdr's pacing: wait prompt → stage text → wait echo →
    bare-Enter submit.
- **`context.go`**: no plugin env relay. `gatherRunContext` builds in-process:
  cwd, `CATS_PANE_ID` (own-pane exclusion; handles both `w1:p3` and `p_<id>`
  forms via `isOwnPane`), workspace = handle prefix, falling back to
  `session.get`'s active workspace; label via `workspace.list`.
- **`drop.go`**: existing-pane drop = `send_input(pane, prompt, submit)` +
  best-effort `agent.focus` (crosses workspaces/tabs, unlike `pane.focus`);
  new-session drop = `tab.create` → `tab.rename` (label "cmd: title") →
  `runCommand` → `waitForAgentReady` → `send_input`. No workspace resolution
  step — `tab.create` lands in the active workspace (where the manager runs).
- **`ui.go` buildTargets**: agent panes straight from `pane.list` meta; agent
  state surfaced in the row desc (`[idle] /path`); "(this project)" via handle
  prefix == own workspace.
- **`main.go`/`launch.go`**: bare `cats-todo` = the manager TUI in the current
  pane; `add`, `version`, `help` subcommands. No `todo`/`todo-ui` split.
- **Stores**: `.cats-todo/todos.json` (project), `~/.config/cats-todo/`
  (global; precedence `CATS_TODO_CONFIG_DIR` > `XDG_CONFIG_HOME` > `~/.config`).
