## Pre-work: orchestration client surface (Explore agent findings, load-bearing)

- **No client type exists** in `internal/orchestration` ‚Äî a client dials the unix socket
  itself and uses the symmetric codec `WriteMessage`/`ReadMessage` (`protocol.go:565/:584`,
  `[u32-LE len][JSON]`, `MaxFrameSize` 8 MiB). The codec + message structs are **untagged**;
  only `host.go`/`cmd/termhost` need `-tags ghostty`.
- Handshake: send `NewHello()` ‚Üí read `Welcome{ProtocolVersion, Error, Panes []uint32}`.
  `welcome.panes` lists surviving pane ids; daemon auto-replays each pane's state after
  hello, but `NewRequestResync(id)` per pane removes the registration race.
- Œ≤ carries **raw input only** (`Input.Data`, base64) ‚Äî no structured key/mouse messages.
  The gateway must encode via `inputenc` before forwarding (exactly why D4 lives server-side).
- Daemon accepts clients **serially** (single-writer); on detach it drops events (no
  buffering) ‚Äî the next client resyncs. Persistent mode: `termhost -socket X -persistent`
  (default socket `/tmp/herdr-termhost.sock`, idle reaper 10m when no client attached).
- Daemon-side cadence: frames coalesce at 16ms (`host.go:25`); `pane_modes` re-checked
  after dirty frames; cwd/title/clipboard from the raw-stream scanner; agent from detectPump.
- Best client exemplar: `internal/orchestration/host_test.go` (hello ‚Üí create ‚Üí event loop).
  `cmd/gateway`/`cmd/smoke`/`cmd/wsprobe` target the **Rust herdr socket** (Œ≥) ‚Äî not termhost.

## 4.1 ‚Äî `cmd/gateway2` (`-tags ghostty`; untagged `stub.go` keeps `go build ./...` green)

- **Model** (pure, built before any daemon connection): one `workspace.Workspace`, one tab,
  fixed two-pane **horizontal** split. `modelSpawner` is a no-op seam impl ‚Äî daemon pane
  creation happens in reconcile, not in Spawn (so the model exists before/without a daemon).
  Pane ids are deterministic in a fresh process (layout's global counter: 1, 2), which is
  what makes gateway-restart adoption line up with `welcome.panes` without persistence.
- **Daemon manager** (`daemon.go`): dial-forever loop with 1s‚Üí5s backoff; hello/welcome
  version check; then **reconcile**: model pane in `welcome.panes` ‚Üí `resize` (enforce model
  grid) + `request_resync`; missing ‚Üí `create_pane` (cwd, cell px, clears `exited`); daemon
  pane not in model ‚Üí `close_pane`. Event pump dispatches: `pane_frame` ‚Üí per-connection
  `FrameTranslator.Translate` fan-out; `pane_modes` ‚Üí `inputModesFrom(Œ≤)` ‚Üí
  `Encoder.SetModes` + `ModesFrom` broadcast; title/cwd/agent cached on the pane model
  (replayed to newly connected browsers) + broadcast; clipboard/exited/error forwarded.
- **Browser sessions** (`gateway.go`): init/welcome handshake (¬ß2; version-mismatch welcome
  carries `error`), then initial state push: layout, cached per-pane modes/title/cwd/agent/
  exited, agents rollup, then `request_resync` per pane (fulls arrive via the pump strictly
  after, so ordering holds). Per-connection writer goroutine drains a 512-msg channel (only
  WSConn writer); overflow drops the connection. Up messages: `key`/`paste`/`raw` ‚Üí focused
  pane; `mouse` ‚Üí addressed pane (encoder `nil` on wheel ‚áí `scroll_viewport` fallback ‚Äî
  inputenc contract); `resize` ‚Üí new area, `applyLayout` (resizes daemon panes whose inner
  grid changed) + layout broadcast; commands: `pane.focus`, `scroll`; everything else
  answers `cmd_result{ok:false}`.
- **Chrome strip**: gateway post-processes `BuildLayout` ‚Äî every pane's inner rect loses
  `chromeRows`(=1) at the top; daemon panes are sized to the **inner** rect. The strip is
  where the browser renders chrome-as-data.
- Lock order: `gateway.mu` ‚Üí `daemon.mu` everywhere (handshake/input under g.mu call
  daemon.send; pump takes g.mu per event). Encoders serialized under g.mu.

## 4.2 ‚Äî `cmd/gateway2/web/index.html`

Per-pane `<div>` (chrome strip + canvas) positioned from layout rects √ó measured cell
metrics (D3); packed-u32 ‚Üí `rgb()` with per-pane `def_fg`/`def_bg` fallback and 0 ‚Üí theme
default (D2); ratatui modifier bits incl. REVERSED/DIM(blend)/HIDDEN; cursor drawn only on
the focused pane. Input: structured `key` events (code/key/mods/kind incl. keyup ‚Äî server
drops unwanted releases), `paste` via Cmd+V/paste event, per-pane mouse in **inner-rect
0-based cell coords** (encoder adds the +1), move deduped per cell, wheel ‚Üí `mouse` when
`modes.mouse || alt_screen` else `cmd scroll` (alternate-scroll needs the wheel to reach the
encoder ‚Äî deliberate deviation from spec ¬ß6's letter, honoring its intent). Click on
unfocused pane sends `cmd pane.focus` first. No VT bytes, no key table ‚Äî Œ±'s JS encoding is
fully dead in this page. Chrome (pub id, title, cwd, agent:state, exit code) is plain text.

## Probe ‚Äî `cmd/wsprobe2` (untagged, stdlib-only; for 4.3)

RFC6455 client (pattern-matched from `cmd/wsprobe`) speaking browserproto: folds
`pane_frame`/`pane_diff` into per-pane grids and runs a semicolon-separated op script:
`wait:MS`, `focus:PANE`, `type:TEXT` (rune ‚Üí W3C code/key/mods map), `key:CODE[:csam]`,
`mouse:PANE:X:Y[:BTN]` (down+up), `wheel`, `scrollcmd`, `expect:PANE:TEXT` (poll),
`absent`, `modes:PANE:mouse|nomouse`, `dump:PANE`. Exits non-zero on failure.
